# ₊⊹ ✴︎ Zentria CLI — Release v0.0.1

> **Fecha:** 10–11 de marzo de 2026  
> **Tag:** `0.0.1`  
> **Estado:** Estable (base)

---

## Resumen

Primera versión funcional de Zentria CLI. Establece la arquitectura base de la aplicación: autenticación, navegación entre modos y escaneo de equipos. Toda la lógica reside en código fuente TypeScript/React, sin empaquetado como ejecutable.

---

## Cambios Incluidos

### ✦ Estructura inicial del proyecto
- Setup de proyecto Node.js con TypeScript 5, React 18 e **Ink 4** (React para terminales).
- Configuración de `tsconfig.json` y estructura de carpetas `source/`.
- Archivo `GEMINI.md` con instrucciones para el agente de IA del proyecto.

### ✦ Autenticación
- Componente `LoginView.tsx` con formulario de usuario y contraseña.
- Navegación con Tab entre campos de input.
- Login contra la API de Zentria ERP mediante `auth.service.ts`.
- Persistencia del token de sesión con la librería **Conf**.

### ✦ Modos de operación
- `AppMode`: selector entre modo **Online** (con API) y **Offline** (local).
- Componente `MainMenuView.tsx` como hub central de navegación.
- Componente `Menu.tsx` reutilizable con navegación por teclado (↑↓ + Enter).

### ✦ Módulos funcionales
- **`OnlineTicketModule.tsx`** — Escáner por lotes: asocia equipos a una `branch` y permite imprimir etiquetas.
- **`GlobalScannerModule.tsx`** — Escáner global sin filtro de branch.
- **`OfflineTicketModule.tsx`** — Escaneo local sin conexión para pruebas.

### ✦ Servicios
- **`api.service.ts`** — Cliente HTTP (Axios) con manejo de errores de sesión y servidor.
- **`auth.service.ts`** — Gestión de credenciales, tokens y configuración persistente.
- **`print.service.ts`** — Generación de etiquetas HTML → PDF (Puppeteer) e impresión térmica.

### ✦ Refactoring de arquitectura
- Separación clara de responsabilidades: componentes, módulos, servicios, tipos.
- `api.types.ts` con interfaces del dominio (equipos, sucursales, respuestas API).
- `useCommand.ts` hook para comandos internos de la CLI.

## Archivos Clave

| Archivo | Descripción |
|---------|-------------|
| `source/app.tsx` | Shell principal, maneja estado de autenticación y modo |
| `source/cli.tsx` | Entry point: configura terminal e inicia React/Ink |
| `source/services/auth.service.ts` | Persistencia de sesión con Conf |
| `source/services/api.service.ts` | HTTP client con manejo de errores |
| `source/modules/online/TicketModule.tsx` | Flujo completo de revisión por lote |

---

## Limitaciones conocidas

- Sin ejecutable portable: requiere Node.js instalado en el equipo destino.
- Sin temas visuales (solo colores hardcodeados).
- Sin modo de salida limpia desde submenús con Escape.

---

## Cómo ejecutar

```bash
npm install --legacy-peer-deps
npm run build
node dist/cli.js
```
