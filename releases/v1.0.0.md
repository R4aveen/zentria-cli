# ₊⊹ ✴︎ Zentria CLI — Release v1.0.0

> **Fecha:** 11 de marzo de 2026  
> **Tag:** `v1.0.0`  
> **Estado:** Estable

---

## Resumen

Versión mayor que transforma Zentria CLI en un **ejecutable portable `.exe`** sin dependencias externas. Se introduce el pipeline de build completo con esbuild + Node.js SEA, se agregan 8 temas visuales, se mejora la experiencia de navegación y se incluye soporte para inyección de variables de entorno en tiempo de build.

---

## Cambios Incluidos

### ✦ Ejecutable portable (Node.js SEA)  
**Commits:** `79c7b5a`, `88bcbdd`

- Pipeline de build en `scripts/build-exe.mjs`:
  1. **esbuild** — Bundlea `source/cli.tsx` en ESM con minificación y shims de `createRequire`.
  2. **Node.js SEA** — Genera blob binario (`sea-prep.blob`) con la app y assets embebidos.
  3. **postject** — Inyecta el blob en una copia de `node.exe`.
- Wrapper CJS (`scripts/sea-entry.cjs`) como punto de entrada SEA: extrae assets embebidos a `%TEMP%` y lanza el bundle ESM.
- Assets embebidos:
  - `bundle.mjs` — App completa
  - `yoga.wasm` — Motor de layout de Ink
  - `logo_etiqueta.png` — Logo para etiquetas de impresión
- Agregado `npm run build:exe` en `package.json`.
- El resultado es `build/zentria-cli.exe` (~83 MB), ejecutable sin Node.js instalado.

### ✦ Inyección de variables de entorno en build-time  
**Commit:** `5685dae`

- El script `build-exe.mjs` lee `.env.production` y reemplaza `process.env['KEY']` en todos los archivos fuente antes de empaquetar (plugin esbuild personalizado `inject-env`).
- Evita exponer credenciales en el entorno del sistema del usuario final.
- `auth.service.ts` carga `.env` solo en modo dev; en producción los valores ya están incrustados.

### ✦ Icono y metadatos del ejecutable  
**Commit:** `71f5bc5`

- Integración de **resedit** para modificar el binario PE de Windows post-build.
- Aplica `public/favicon.ico` como icono de la ventana y del explorador de archivos.
- Metadatos de versión en `VERSIONINFO`: ProductName, FileDescription, CompanyName, LegalCopyright, FileVersion, ProductVersion.

### ✦ 8 temas visuales

- Componente `ThemeSelector.tsx` con selector interactivo adaptativo al tamaño de terminal.

### ✦ Mejoras de UX y navegación  
**Commits:** `fe1debb`, `5685dae`

- `useTerminalSize` hook para detección de redimensionado de ventana y re-render reactivo.
- Pantalla alternativa (`alternate screen buffer`) en `cli.tsx` para no contaminar el historial de la terminal.
- Título de ventana de terminal personalizado.
- Navegación con Escape para volver de submenús.
- Funcionalidad de salida limpia desde el menú principal.
- `OnlineTicketModule` muestra cantidades recibidas vs esperadas por equipo.

### ✦ Scripts de build multiplataforma  
**Commit:** `5685dae`

- `scripts/build.ps1` — Script PowerShell con modos: `dev`, `bundle`, `exe`, `all`.
- `scripts/build.sh` — Equivalente para Linux/macOS.

### ✦ Corrección EBUSY  
**Commit:** `88bcbdd`

- Si `zentria-cli.exe` estaba bloqueado por un proceso activo, el build fallaba con `EBUSY`.
- Solución: antes de copiar `node.exe`, intentar `unlinkSync`; si falla, `renameSync` a `.old` para desbloquearlo.

## Cómo buildear

```bash
# Instalar dependencias
npm install --legacy-peer-deps

# Generar ejecutable
node scripts/build-exe.mjs
# o con PowerShell:
.\scripts\build.ps1 exe
```

> **Nota:** Matar cualquier proceso `zentria-cli.exe` activo antes de buildear para evitar el error EBUSY.

## Limitaciones conocidas

- El `.exe` activa alertas de **Windows Defender / SmartScreen** al no tener certificado de código EV.
- Sin firma digital en el ejecutable.
- Distribución requiere que el usuario tenga Chrome instalado (impresión de etiquetas).
